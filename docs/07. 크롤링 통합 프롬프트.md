# 크롤링 통합 프롬프트

## 목표

데이터 형태를 모르는 상태에서도 정적 웹페이지, `HTML table`, `XML`, `JSON`, `Excel` 중 어떤 방식으로 처리해야 하는지 판단하고, 최종적으로 `CSV` 파일을 생성한다.

처음 크롤링을 맡은 비전공자가 그대로 복사해서 사용할 수 있는 요청 프롬프트와 작업 기준을 정리한다.

이 문서는 아래 상황에서 사용한다.

```text
데이터 출처는 있는데 어떤 방식으로 가져와야 할지 모를 때
→ 데이터 형태를 먼저 판단한다
→ 형태별 문서를 참고한다
→ 최종 결과를 CSV로 통일한다
```

## 전체 작업 흐름

```text
데이터 출처 확인
→ 데이터 형태 판단
→ 형태에 맞는 수집 방식 선택
→ 원본 데이터 저장
→ pandas DataFrame 변환
→ 컬럼명/결측치/중복값 정리
→ CSV 저장
→ 출처와 수집 날짜 기록
```

## 1. 데이터 형태 판단 기준

### 정적 웹페이지

브라우저에서 보이는 글자나 링크가 HTML 안에 바로 들어있는 페이지다.

확인 방법:

- 페이지에서 우클릭 후 `페이지 소스 보기`를 한다.
- 찾고 싶은 텍스트를 검색했을 때 소스 안에서 바로 보이면 정적 페이지일 가능성이 높다.

사용 도구:

```text
requests
BeautifulSoup
pandas
```

참고 문서:

```text
docs/02. 정적페이지 크롤링.md
```

### HTML table

웹페이지 안에 표가 있고, HTML의 `<table>` 태그로 만들어진 데이터다.

확인 방법:

- 개발자 도구에서 `<table>`, `<tr>`, `<th>`, `<td>` 태그가 있는지 확인한다.
- `pandas.read_html(url)`로 표가 읽히는지 확인한다.

사용 도구:

```text
pandas.read_html()
```

참고 문서:

```text
docs/03. HTML table CSV 변환.md
```

### XML

태그가 반복되는 구조의 데이터다.

예시:

```xml
<item>
  <name>서울</name>
  <value>10000</value>
</item>
```

확인 방법:

- URL에 접속했을 때 `<item>`, `<row>`, `<data>` 같은 태그가 보이는지 확인한다.
- 공공데이터 API에서 응답 형식이 XML인지 확인한다.

사용 도구:

```text
requests
BeautifulSoup(xml)
pandas
```

참고 문서:

```text
docs/04. XML CSV 변환.md
```

### JSON

`{}`와 `[]`로 이루어진 API 데이터다.

예시:

```json
{
  "items": [
    {"region": "서울", "count": 10000}
  ]
}
```

확인 방법:

- URL에 접속했을 때 `{}` 또는 `[]` 구조가 보이는지 확인한다.
- API 문서에서 응답 형식이 JSON인지 확인한다.

사용 도구:

```text
requests
response.json()
pandas.json_normalize()
```

참고 문서:

```text
docs/05. JSON CSV 변환.md
```

### Excel

파일 확장자가 `.xlsx` 또는 `.xls`인 데이터다.

확인 방법:

- 공공기관 페이지에서 Excel 파일을 다운로드한다.
- 파일 안에 시트가 여러 개 있는지 확인한다.

사용 도구:

```text
pandas.ExcelFile()
pandas.read_excel()
```

참고 문서:

```text
docs/06. Excel CSV 변환.md
```

## 2. 공통 폴더 구조

크롤링 데이터는 원본과 전처리 결과를 분리해서 저장한다.

```text
data/
├── raw/          # 원본 데이터 저장
├── processed/    # 전처리 완료 CSV 저장
└── README.md     # 데이터 출처, 수집 날짜, 설명 기록
```

### raw 폴더

`data/raw/`에는 원본 파일을 저장한다.

예시:

```text
data/raw/source.xlsx
data/raw/source.json
data/raw/source.xml
```

원본 데이터는 가능하면 수정하지 않는다.

### processed 폴더

`data/processed/`에는 전처리 완료된 CSV 파일을 저장한다.

예시:

```text
data/processed/ev_registration.csv
data/processed/ev_charger_status.csv
```

### README.md

`data/README.md`에는 데이터 출처와 수집 날짜를 기록한다.

나중에 발표나 보고서 작성 시 데이터 근거를 설명하기 위해 필요하다.

## 3. 공통 전처리 체크리스트

데이터 형태가 달라도 `DataFrame`으로 만든 뒤 확인해야 할 내용은 거의 같다.

```python
print(df.head())
print(df.info())
print(df.isnull().sum())
print(df.duplicated().sum())
```

### 한 줄씩 설명

```python
print(df.head())
```

데이터 앞부분 5줄을 확인한다.

```python
print(df.info())
```

컬럼명, 데이터 개수, 데이터 타입을 확인한다.

```python
print(df.isnull().sum())
```

컬럼별 비어 있는 값 개수를 확인한다.

```python
print(df.duplicated().sum())
```

중복 행 개수를 확인한다.

## 4. 공통 전처리 코드

```python
df.columns = df.columns.astype(str).str.strip()
df = df.dropna(how="all")
df = df.drop_duplicates()

for column in df.select_dtypes(include="object").columns:
    df[column] = df[column].astype(str).str.strip()
```

### 한 줄씩 설명

```python
df.columns = df.columns.astype(str).str.strip()
```

컬럼명을 문자열로 바꾸고 앞뒤 공백을 제거한다.

```python
df = df.dropna(how="all")
```

모든 값이 비어 있는 행을 제거한다.

```python
df = df.drop_duplicates()
```

완전히 같은 중복 행을 제거한다.

```python
for column in df.select_dtypes(include="object").columns:
```

문자열 컬럼만 하나씩 선택한다.

```python
df[column] = df[column].astype(str).str.strip()
```

문자열 데이터의 앞뒤 공백을 제거한다.

예시:

```text
" 서울 " → "서울"
```

## 5. 공통 CSV 저장 코드

```python
df.to_csv("data/processed/result.csv", index=False, encoding="utf-8-sig")
```

### 한 줄씩 설명

```python
df.to_csv(...)
```

`DataFrame`을 CSV 파일로 저장한다.

```python
"data/processed/result.csv"
```

저장할 파일 경로다.

```python
index=False
```

CSV에 `0, 1, 2` 같은 pandas 인덱스를 저장하지 않는다.

```python
encoding="utf-8-sig"
```

엑셀에서 한글이 깨지는 문제를 줄이기 위한 인코딩이다.

## 8. 전체 작업 요청용 프롬프트

```text

목표:
- 데이터 형태가 정적 웹페이지, HTML table, XML, JSON, Excel 중 무엇인지 먼저 판단해줘.
- 왜 그렇게 판단했는지도 설명해줘.
- 형태에 맞는 파이썬 수집 코드를 작성해줘.
- 코드에는 비전공자도 이해할 수 있게 단계별 주석을 달아줘.
- 각 변수에 어떤 데이터가 들어가는지 예시를 보여줘.
- 수집한 데이터를 pandas DataFrame으로 변환해줘.
- 컬럼명 정리, 결측치 확인, 중복 제거, 데이터 타입 확인까지 해줘.
- 최종 결과는 CSV 파일로 저장해줘.
- CSV 저장 시 한글이 깨지지 않도록 utf-8-sig 인코딩을 사용해줘.
- 실행 중 오류가 날 수 있는 부분은 원인과 해결 방법도 같이 설명해줘.

저장 규칙:

- 전처리 완료 CSV는 data/processed/ 폴더에 저장

데이터 출처:
- 여기에 URL 또는 파일 경로 입력

수집하고 싶은 컬럼:
- 여기에 필요한 컬럼 입력

최종 CSV 파일명:
- 예: ev_charger_status.csv
```

## 15. 크롤링 주의사항

- 사이트의 이용약관과 `robots.txt`를 확인한다.
- 너무 짧은 시간에 많은 요청을 보내지 않는다.
- API 키가 필요한 경우 `.env` 파일에 따로 보관한다.
- 원본 URL, 수집 날짜, 수집 기준을 반드시 기록한다.
- 원본 데이터와 전처리 데이터를 구분해서 저장한다.
- 데이터 컬럼 의미를 모르면 임의로 해석하지 말고 출처 문서를 확인한다.
- 개인정보나 민감정보가 포함된 데이터는 수집하지 않는다.

## 초보자가 꼭 기억할 것

- 데이터 형태를 먼저 판단해야 한다.
- 어떤 형태든 최종 목표는 `DataFrame`을 만든 뒤 CSV로 저장하는 것이다.
- 원본 데이터는 보존하고 전처리 결과는 따로 저장한다.
- 한글 CSV는 `utf-8-sig`로 저장한다.
- 출처와 수집 날짜를 기록해야 발표나 보고서에서 근거를 설명할 수 있다.
