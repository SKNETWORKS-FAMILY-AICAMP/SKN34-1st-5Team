## dotenv 패키지 설명

`dotenv`는 `.env` 파일에 적어둔 환경변수를 Python 코드에서 읽을 수 있게 도와주는 패키지다.

이 프로젝트에서는 DB 접속 정보처럼 코드에 직접 적으면 안 되는 값을 `.env` 파일로 분리해서 관리한다.

예시:

```env
MYSQL_PORT=3306
MYSQL_ROOT_PASSWORD=root_password
DB_NAME=skn34_project
DB_USER=skn34_user
DB_PASSWORD=skn34_password
```

Python 코드에서는 `python-dotenv` 패키지의 `load_dotenv()`를 사용한다.

```python
from dotenv import load_dotenv

load_dotenv()
```

`load_dotenv()`가 실행되면 `.env` 파일의 값들이 환경변수로 등록된다. 이후에는 `os.getenv()`로 값을 가져올 수 있다.

```python
import os

db_name = os.getenv("DB_NAME")
db_user = os.getenv("DB_USER")
db_password = os.getenv("DB_PASSWORD")
```

주의할 점:

- 실제 사용하는 패키지는 `python-dotenv`이다.
- 코드에서는 `from dotenv import load_dotenv` 형태로 import한다.
- `.env` 파일에는 비밀번호가 들어갈 수 있으므로 Git에 올리지 않는다.
- 팀원에게 공유할 기본 형식은 `.env.example`에 작성한다.

## 패키지 버전 갱신
```bash
pip freeze > requirements.txt
```

`pip freeze > requirements.txt`를 실행하면 현재 가상환경에 설치된 패키지 목록으로 `requirements.txt`가 덮어써진다.

따라서 필요한 패키지가 빠지지 않도록 먼저 설치 상태를 확인해야 한다.

```bash
pip install python-dotenv PyMySQL mysql-connector-python
pip freeze > requirements.txt
```
